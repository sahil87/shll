- [ ] [38a6] 2026-06-13: Bake HOMEBREW_NO_REQUIRE_TAP_TRUST=1 into shll's brew install/upgrade invocations as a temporary workaround for a Homebrew 6.0 Linux-sandbox bug. ROOT CAUSE (diagnosed 2026-06-12 on idea v0.0.9->0.0.10 upgrade; not a shll or tap bug): Homebrew 6.0's Linux bubblewrap sandbox runs the formula build/install in build.rb inside bwrap, where deny_read_home (Library/Homebrew/sandbox.rb ~line 191) masks almost all of $HOME. Its exception list covers HOMEBREW_PREFIX/CACHE/LOGS/TEMP but NOT HOMEBREW_USER_CONFIG_HOME (~/.homebrew), where trust.json lives. So when HOMEBREW_REQUIRE_TAP_TRUST=1 is set, the sandboxed build.rb re-checks tap trust, cannot read ~/.homebrew/trust.json, and raises Homebrew::UntrustedTapError at build.rb:264 — three lines BEFORE the error pipe is connected (build.rb:267), so the message is swallowed and brew surfaces only an opaque 'bwrap ... exited with 1' with a giant arg dump. This breaks 'brew install/upgrade sahil87/tap/<formula>' for EVERY roster tool on any Linux box where the user set HOMEBREW_REQUIRE_TAP_TRUST=1 — which shll itself encourages via 'shll shell-setup --trust-tap' (writes that export line). Verified workaround: prefix the brew call with HOMEBREW_NO_REQUIRE_TAP_TRUST=1 (keeps the sandbox active, skips only the broken in-sandbox trust re-check); 'HOMEBREW_NO_REQUIRE_TAP_TRUST=1 brew upgrade sahil87/tap/idea' completed cleanly. IMPLEMENTATION: inject HOMEBREW_NO_REQUIRE_TAP_TRUST=1 into the environment of shll's brew-spawning subprocesses ONLY (the brew install/upgrade/update calls), not per-tool '<tool> update' delegations or unrelated procs. Call sites are all in internal/proc via exec.CommandContext (proc.go) — Request has NO Env field today (cmd.Env is never set, so children inherit the parent env). Add an 'Env []string' (or map) field to proc.Request and have defaultRunner append it to cmd.Environ(); then set it on the brew install/upgrade/update Requests. The brew foreground calls live at: install.go:145 (brew install <formula>), update.go:242 (brew update --quiet), update.go:295 (brew upgrade shll self), and update.go:417/upgradeArgv -> upgradeTool at update.go:404 (brew upgrade <formula> fallback).
